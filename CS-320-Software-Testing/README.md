
# CS 320 Software Testing, Automation, and Quality Assurance

This repository contains my portfolio artifacts for CS 320. The files include my Contact Service code and unit tests from Project One, as well as my Project Two summary and reflections report.

## How can I ensure that my code, program, or software is functional and secure?

I can ensure that my software is functional and secure by testing it against clear requirements. In this project, I used JUnit tests to check valid and invalid inputs for the Contact Service. This helped confirm that the program handled errors correctly instead of accepting unsafe data.

## How do I interpret user needs and incorporate them into a program?

I interpret user needs by turning requirements into specific rules in the code. For example, the Contact object needed limits for ID, name, phone number, and address. These needs became validation checks and unit tests. This made the program match the expected behavior more closely.

## How do I approach designing software?

I approach software design by separating responsibilities. The Contact class stores valid contact data, while ContactService manages adding, deleting, and updating contacts. This structure made the code easier to test, understand, and maintain.
